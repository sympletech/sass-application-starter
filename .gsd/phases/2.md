# Phase 2: Component Extraction & Cleanup

**Objective**: Systematically break down large pages, extract reusable UI elements, and remove inline styles in favor of Tailwind classes and modular CSS.

## Tasks

### 1. Extract Shared Hooks
Create a centralized hook for common landing page logic.
- `src/hooks/useReveal.js`: Extract the `IntersectionObserver` logic used in `Home.jsx` and `About.jsx`.

### 2. Extract Reusable Components
Move page-specific components to the global `src/components` directory to improve reusability and project structure.
- **FeatureCard**: Move from `Home.jsx` to `src/components/FeatureCard/FeatureCard.jsx`.
- **PricingCard**: Move from `pages/home/sub-components/` to `src/components/PricingCard/PricingCard.jsx`.
- **Layout Components**: Create an `AppSection.jsx` component to handle standard section padding (`80px 0`) and the `reveal` class, reducing boilerplate.

### 3. Refactor Pages (Removal of Inline Styles)
For each page, replace `style={{ ... }}` props with Tailwind utility classes or named CSS files.
- **Home.jsx**:
  - Replace Hero section inline styles.
  - Integrate `useReveal` hook and `AppSection` component.
  - Convert image transforms and positioning to Tailwind + CSS Classes if possible.
- **About.jsx**:
  - Replace mission and story section styling.
  - Integrate `useReveal` hook.
- **Legal.jsx**:
  - Refactor Tabs and Card container styling.

### 4. Modular CSS Transition
- Create `.css` files for any component that still needs complex styling not easily handled by Tailwind utilities alone.
- Ensure all new components follow the `REFACTORING_STANDARDS.md`.

## Execution
Run `/execute 2` to start.
