# User Preferences

> **Purpose**: Records how the user likes things done. Helps agents match the user's working style and expectations.
> **Last Updated**: [Not yet populated]

---

## How to Use This Document

1. **When starting a task**: Check for relevant preferences
2. **After receiving feedback**: Update preferences based on what the user corrected
3. **When unsure how to proceed**: Check if there's a documented preference

---

## Communication Preferences

### Question Asking
- [ ] Ask all clarifying questions upfront before starting
- [ ] Ask questions as needed during implementation
- [ ] Minimize questions, make reasonable assumptions

### Progress Updates
- [ ] Provide detailed progress updates
- [ ] Brief status updates only
- [ ] Only report when complete or blocked

### Explanations
- [ ] Explain reasoning in detail
- [ ] Brief explanations only
- [ ] Just do the work, explain if asked

---

## Code Preferences

### Code Comments
- [ ] Extensive inline comments
- [ ] Comments for complex logic only
- [ ] JSDoc for functions, minimal inline comments
- [ ] Minimal comments (self-documenting code)

### Variable Naming
- [ ] Descriptive (longer) names preferred
- [ ] Concise names preferred
- [ ] Match existing codebase style

### Error Handling
- [ ] Defensive - handle all edge cases
- [ ] Balanced - handle likely issues
- [ ] Minimal - happy path focus

---

## Workflow Preferences

### Task Breakdown
- [ ] Break into very small steps
- [ ] Medium-sized logical chunks
- [ ] Complete features in one go

### Testing
- [ ] Write tests first (TDD)
- [ ] Write tests after implementation
- [ ] Manual testing is sufficient

### Documentation
- [ ] Update docs immediately with changes
- [ ] Update docs at end of task
- [ ] Docs only when specifically requested

---

## Technology Preferences

### Third-Party Libraries
- [ ] Prefer well-maintained libraries over custom code
- [ ] Minimize dependencies
- [ ] Case-by-case evaluation

### New Features
- [ ] Use latest stable features
- [ ] Conservative - proven patterns only
- [ ] Match what's already in codebase
