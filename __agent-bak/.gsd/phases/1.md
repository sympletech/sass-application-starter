# Phase 1: Foundation & Tailwind Integration

**Objective**: Install Tailwind CSS v3, configure content paths, and establish the refactoring patterns.

## Tasks

### 1. Install Dependencies
Install Tailwind CSS and its peer dependencies as dev dependencies in the root project (since it manages the client build).
```bash
npm install -D tailwindcss postcss autoprefixer
```

### 2. Initialize Configuration
Generate `tailwind.config.js` and `postcss.config.js` at the project root.
```bash
npx tailwindcss init -p
```

### 3. Configure Paths & Theme
Update `tailwind.config.js` to:
- Scan all files in `@client/src` for classes.
- Extend the theme to map existing CSS variables from `theme.css`.

**Configuration Plan:**
```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./@client/src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {
      colors: {
        brand: {
          500: 'var(--color-brand-500)',
          600: 'var(--color-brand-600)',
          // ... map other brand colors
        },
        surface: {
          base: 'var(--surface-base)',
          muted: 'var(--surface-muted)',
        },
        text: {
          strong: 'var(--text-strong)',
          body: 'var(--text-body)',
        }
      },
      // Extend other properties as needed
    },
  },
  plugins: [],
}
```

### 4. Inject Tailwind Directives
Modify `@client/src/index.css` to include the Tailwind directives at the very top.
```css
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Existing styles follow... */
```

### 5. Create Refactoring Standards
Create `.gsd/REFACTORING_STANDARDS.md` to document:
- **Naming**: Use `kebab-case` for new component files.
- **Styling**: Prefer Tailwind utility classes over inline styles.
- **Theming**: Use `bg-brand-500` instead of hex codes.
- **Components**: Break down large components into functional atoms in `@client/src/components`.

### 6. Verification
- Run `npm start`.
- Edit `Home.jsx` (or similar) to add a test Tailwind class (e.g., `text-brand-500`).
- Verify visual output.

## Execution
Run `/execute 1` to start.
